# Nakobi Grill — Restaurant POS System

A full-stack Point of Sale system built for Nakobi Grill. Manages orders, inventory, staff, and the restaurant menu from a single web-based portal, with role-based access control so each staff member only sees what they need.

---

## Features

### Orders
- Take new dine-in or takeaway orders from a live menu modal
- View all active orders on a Kanban-style card grid
- Advance orders through statuses: **Pending → Preparing → Ready**
- Cancel orders with optimistic UI updates
- Order numbers auto-generated by database trigger

### Inventory
- Track ingredient stock levels with current quantity and unit
- Restock items or log stock-outs with full audit trail (inventory logs)
- Status automatically computed by database trigger: **OK / Low / Critical**
- Add and delete inventory items

### Dashboard *(Admin only)*
- Today's revenue, order count, dine-in vs takeaway split
- Active staff count and total staff
- Stock alert summary (critical and low items)
- Recent orders feed (last 5)
- Top-selling menu items for the past 7 days
- Active staff on shift

### Menu Management *(Admin only)*
- Create and delete meal categories
- Add and remove menu items with name, description, price, and category
- Menu is live — changes reflect immediately in the order modal

### Staff / Users *(Admin only)*
- View all staff members (excluding the logged-in admin)
- Create new staff accounts with role, shift type, and status
- Edit staff details
- Toggle active/inactive status
- Delete staff accounts (cascades to auth)

### Settings
- Profile tab: view your name, email, role, shift, status, and join date
- Menu tab: manage categories and menu items *(Admin only)*

### Authentication & Access Control
- Supabase Auth with email/password login
- Role-based routing: each role redirects to their home page on login
  - **Admin** → Dashboard
  - **Inventory Manager** → Inventory
  - **Order Taker** → Orders
- Middleware enforces route protection on every request
- Portal layout performs a server-side auth check as a second guard
- Unauthenticated users are always redirected to `/login`
- Sidebar navigation filtered to only show links the logged-in role can access

## Getting Started

### Prerequisites

- Node.js 18+
- Yarn
- A [Supabase](https://supabase.com) project

### Environment Variables

Create a `.env.local` file in the project root:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
```

> The service role key is used server-side only (in Next.js Server Actions) for admin operations like creating and deleting auth users.

### Database Setup

Run the SQL in `Database_Schema.md` against your Supabase project to create the tables, triggers, and RLS policies.

### Install & Run

```bash
yarn install
yarn dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

### Build for Production

```bash
yarn build
yarn start
```

## Database Overview

| Table             | Purpose                                              |
|-------------------|------------------------------------------------------|
| `profiles`        | Staff accounts linked to Supabase Auth users         |
| `menu_categories` | Meal categories shown in the order modal             |
| `menu_items`      | Individual menu items with price and category        |
| `orders`          | Order headers (type, status, total, order number)    |
| `order_items`     | Line items per order (qty, unit price, item name)    |
| `inventory_items` | Ingredient stock with auto-computed status           |
| `inventory_logs`  | Audit log for every restock and stock-out event      |

Key database triggers:
- `on_auth_user_created` — creates a profile row when a new auth user is created
- `trg_inventory_status` — auto-sets inventory status to ok/low/critical based on thresholds
- `trg_order_number` — generates a sequential order number on insert
- `trg_order_total` — computes the order total from line items on insert/update
- `trg_orders_updated_at` — keeps the `updated_at` timestamp current
